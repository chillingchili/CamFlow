# Plan 02-01: Backend API — Presets + PTZ Movement

## Objective

Add REST API endpoints for:
1. Preset CRUD: GET all presets, PUT update preset (name, ptz_number, active, settle_time), PATCH reorder presets
2. PTZ movement: POST /api/ptz/move (direction + speed), POST /api/ptz/zoom (in/out + speed), POST /api/ptz/stop

All endpoints are JWT-protected. Preset updates persist to SQLite. PTZ movement commands relay through the WebSocket tunnel to the agent via the existing command relay in `routes/commands.ts`.

## Prerequisites

- Phase 1 backend (Express + JWT auth + WebSocket tunnel) must be complete
- SQLite schema already has presets table with: name, ptz_number, active, sort_order, settle_time
- Existing command relay infrastructure in `routes/commands.ts`

## Tasks

### Task 1: Create preset management API endpoints

**TDD:** Write failing tests first, then implement.

**Test file:** `cloud/server/src/__tests__/presets.test.ts`

Tests to write:
- `GET /api/presets` — returns all 8 presets ordered by sort_order, JWT required

- PATCH /api/presets/reorder: Transaction wrapping UPDATE for each {id, sort_order} pair

**Wire into:** `cloud/server/src/index.ts` — add `app.use('/api/presets', authenticateToken, presetsRouter)`

**Verify:** All tests pass. Presets persist across server restarts.
