# Plan 02-02: Frontend — Setup/Live Tabs + Preset Grid + Joystick

## Objective

Transform the single-page dashboard into a polished two-tab interface (Setup | Live) with:
1. **Tab navigation** — top tab bar (Live | Setup + gear icon), Setup as default after login
2. **Preset grid** — vertical card list with inline name editing, toggle switches, drag reorder (grip handle), per-preset settle time input
3. **PTZ joystick** — cross D-pad layout, press-and-hold movement, side-by-side zoom buttons, 1-100 speed slider, keyboard shortcuts
4. **Camera switcher** — polished large buttons always visible in Live tab
5. **Settings panel** — gear icon opens panel with system info

All interactions use the Phase 1 ACK feedback pattern (optimistic → solid on ACK → red flash + toast on NACK).

## Prerequisites

- Phase 1 frontend complete (StatusBar, Toast, useCommandState, useWebSocket, api.ts)
- Plan 02-01 backend endpoints available (GET/PUT/PATCH presets, POST ptz/move, POST ptz/zoom, POST ptz/stop)
- ACK pipeline from Phase 1 working

**Implementation:**

**`cloud/client/src/components/Joystick.tsx`**:
```
// Cross D-pad layout:
//         [Up W]
// [Left A] [Stop] [Right D]
//         [Down S]
// Zoom In [T] / Zoom Out [R] to the right of D-pad
// Speed slider 1-100 below the D-pad
// 
// Interaction:
//   onPointerDown → send move command (POST /api/ptz/move)
//   onPointerUp / onPointerLeave → send stop command (POST /api/ptz/stop)
//   Speed slider onChange → update local speed state
//
// Keyboard (via useKeyboard hook):
//   WASD → trigger same move commands as D-pad buttons
//   R/T → trigger zoom commands
//   F/G → adjust d-pad speed by ±1
//   V/B → adjust zoom speed by ±1
```
