      - `on('open')` — Clear reconnect backoff. Call onConnected callback. Start heartbeat interval (5s). Replay any queued commands with timestamp check (reject commands >15s old).
      - `on('message')` — Parse JSON. If `{ type: 'command', ... }`, call the registered command handler. Ignore other message types (health is outbound only).
      - `on('close')` — Start reconnect timer with exponential backoff: `delay = min(1000 * 2^attempt, 30000)` with ±50% jitter. `on('error')` — log and let close handler handle reconnect.
    - `send(message)` — `JSON.stringify` and `ws.send()`. If not connected, return false. Only sends command `ack` responses and `health` messages.
    - `onCommand(callback)` — Register the command handler. Used by router.js.
    - `getConnectionState()` — Return `{ connected: ws.readyState === OPEN, lastHeartbeat }`.

    **Step 4 — Health Reporter (`agent/src/health.js`):**

    - Health is called by tunnel's heartbeat interval (every 5s). The tunnel module sends it over WebSocket.

    **Step 2 — PTZ Controller (`agent/src/ptz.js`):**
    BirdDog PTZ cameras use NDI protocol. NDI SDK is a native C library — for the agent (Node.js), we use NDI's PTZ commands. Since direct NDI SDK bindings in Node.js are complex and undocumented, we use two approaches:
    
    **Primary approach (when NDI Tools installed):** Use BirdDog's HTTP REST API if available. Most BirdDog cameras expose a REST API at `http://<PTZ_IP>/v1/`. Check reachability via HTTP ping.
    
    **Fallback approach:** Shell out to NDI Tools CLI if installed. But this is fragile.
    
    **For Phase 1 implementation:** Create a clean abstraction. The `ptz.js` module exports:
    - `recallPreset(presetNumber)` — Sends the NDI PTZ preset recall command. Implementation: uses the BirdDog REST API `POST http://<PTZ_IP>/v1/ptz/preset/recall` with body `{ preset: presetNumber }`. Returns promise.
    - `savePreset(presetNumber)` — Sends the NDI PTZ preset save command. `POST http://<PTZ_IP>/v1/ptz/preset/save` with body `{ preset: presetNumber }`. Returns promise.
    - `isReachable()` — Pings the PTZ IP. Caches result for 5s.
    
    **Important:** The actual NDI PTZ protocol may differ from REST API depending on BirdDog model. Since we have no physical camera access (per PROJECT.md constraint), implement to the BirdDog documented API spec. Use a configurable `PTZ_PROTOCOL` env var (`rest` or `ndi-direct`). Default to `rest`. This abstraction layer allows swapping the backend later without changing the router.
    
    Each PTZ command has a 5s timeout (per CONTEXT.md). Wrap in Promise.race with timeout.

    **Step 1 — Command Router (`agent/src/router.js`):**
    The router is the central dispatch — receives cloud commands from the tunnel and routes to the correct hardware module, then sends ACK responses back.

    ```javascript
    // agent/src/router.js
    import { setScene } from './obs.js';
    import { recallPreset, savePreset } from './ptz.js';

    const PTZ_TIMEOUT = 5000;  // 5s per CONTEXT.md
    const OBS_TIMEOUT = 2000;  // 2s per CONTEXT.md

    export async function dispatch(command, requestId, sendAck) {
      try {
        switch (command.type) {
          case 'obs_scene': {
            const result = await Promise.race([
              setScene(command.sceneName),
              timeout(OBS_TIMEOUT, 'OBS scene switch timed out')
            ]);
            sendAck({ type: 'ack', requestId, status: 'ok' });
            break;
          }
          case 'ptz_preset_recall': {
            const result = await Promise.race([
              recallPreset(command.presetNumber),
              timeout(PTZ_TIMEOUT, 'PTZ preset recall timed out')
            ]);
            sendAck({ type: 'ack', requestId, status: 'ok' });
            break;
          }
          case 'ptz_preset_save': {
            const result = await Promise.race([
              savePreset(command.presetNumber),
              timeout(PTZ_TIMEOUT, 'PTZ preset save timed out')
            ]);
            sendAck({ type: 'ack', requestId, status: 'ok' });
            break;
          }
          default:
            sendAck({ type: 'ack', requestId, status: 'error', error: `Unknown command type: ${command.type}` });
        }
      } catch (err) {
        sendAck({ type: 'ack', requestId, status: 'error', error: err.message });
      }
    }

    function timeout(ms, message) {
      return new Promise((_, reject) => setTimeout(() => reject(new Error(message)), ms));
    }
    ```

    **Step 2 — Agent Entry Point (`agent/src/index.js`):**
    Wire everything together:
    ```javascript
    import { connect, onCommand, send } from './tunnel.js';
    import { connect as connectOBS } from './obs.js';
    import { dispatch } from './router.js';

    async function start() {
      // Start OBS connection (non-blocking — OBS might not be running yet)
      connectOBS().catch(err => console.error('OBS initial connect failed:', err.message));

      // Start tunnel and register command handler
      onCommand((command, requestId) => {
        dispatch(command, requestId, (ack) => send(ack));
      });

      await connect();
      console.log('CamFlow agent started');
    }

    start().catch(err => {
      console.error('Agent startup failed:', err);
      process.exit(1);
    });
    ```
